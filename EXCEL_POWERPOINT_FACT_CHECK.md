## 1. Excel基礎と実践

### 検証項目1: Excelの基本構造

**【記載内容】**
- セル（Cell）: データを入力する最小単位。列（A, B, C...）と行（1, 2, 3...）の交点で識別（例：A1セル）
- ワークシート（Sheet）: セルが集まった表のこと。1つのファイル（ブック）に複数のシートを作成可能
- ブック（Workbook）: Excelファイル全体のこと。拡張子は「.xlsx」
- 数式（Formula）: 「=」から始まる計算式

**【検証結果】**: ✅ **正確**

**【根拠】**:
- Microsoft公式ドキュメント「Excel の基本」に準拠
- セルは列（A～XFD: 16,384列）と行（1～1,048,576行）の交点で識別
- ワークシートは1つのブックに最大1,048,576行 × 16,384列のデータを格納可能
- .xlsx形式はExcel 2007以降の標準形式（XML形式、Office Open XML）
- 数式は「=」で始まり、関数、演算子、セル参照を含む

**【参考文献】**:
- Microsoft Support: "Excel の基本" (https://support.microsoft.com/ja-jp/excel)
- Microsoft 365 Documentation

### 検証項目3: セル参照の種類

**【記載内容】**
- 相対参照（A1）: 数式をコピーすると、参照先が相対的に移動
- 絶対参照（$A$1）: 数式をコピーしても、参照先が固定
- 複合参照（$A1, A$1）: 列のみ固定、または行のみ固定
- F4キーで切り替え可能

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **相対参照**: デフォルトの参照形式。数式をコピーすると行・列ともに相対的に変化
- **絶対参照**: $記号で列と行を固定（例: $A$1）
- **複合参照**: 
  - $A1: 列Aを固定、行は相対的
  - A$1: 行1を固定、列は相対的
- **F4キー**: 数式バーでセル参照を選択しF4キーを押すと、相対参照→絶対参照→複合参照（列固定）→複合参照（行固定）→相対参照の順に循環

**【参考文献】**:
- Microsoft Support: "セル参照の切り替え" (https://support.microsoft.com/ja-jp/office/dfec08cd-ae65-4f56-839e-5f0d8d0baca9)

## 2. Excel必須関数

### 検証項目4: 基本の集計関数

**【記載内容】**
- SUM: 合計を計算、書式 =SUM(範囲)、ショートカット Alt + Shift + =
- AVERAGE: 平均値を計算、書式 =AVERAGE(範囲)
- COUNT: 数値が入力されているセルの個数をカウント
- MAX / MIN: 最大値・最小値を取得

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **SUM関数**: =SUM(number1, [number2], ...) で合計を計算
  - Alt + Shift + = (または Alt + =): オートSUM機能
- **AVERAGE関数**: =AVERAGE(number1, [number2], ...) で算術平均を計算
- **COUNT関数**: =COUNT(value1, [value2], ...) で数値が含まれるセルの個数を返す
  - COUNTA: 空白以外のセルをカウント（記載通り）
- **MAX関数**: =MAX(number1, [number2], ...) で最大値を返す
- **MIN関数**: =MIN(number1, [number2], ...) で最小値を返す

### 検証項目5: IF関数（条件分岐）

**【記載内容】**
- 書式: =IF(条件, 真の場合の値, 偽の場合の値)
- 例1: =IF(A1>=80, "合格", "不合格")
- 例2（ネスト）: =IF(A1>=90, "優", IF(A1>=80, "良", IF(A1>=70, "可", "不可")))

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **IF関数の構文**: =IF(logical_test, value_if_true, [value_if_false])
  - logical_test: 評価する条件
  - value_if_true: 条件が真の場合の戻り値
  - value_if_false: 条件が偽の場合の戻り値（省略可能）
- **ネスト（入れ子）**: IF関数内に別のIF関数を記述可能
  - Excel 2007以降: 最大64レベルまでネスト可能
- 例示された数式は構文的に正しく、実用的

**【参考文献】**:
- Microsoft Support: "IF 関数" (https://support.microsoft.com/ja-jp/office/69aed7c9-4e8a-4755-a9bc-aa8bbff73be2)

### 検証項目6: AND / OR関数

**【記載内容】**
- AND関数: すべての条件が真の場合にTRUE
  - 例: =IF(AND(A1>=70, B1>=70), "合格", "不合格")
- OR関数: いずれかの条件が真の場合にTRUE
  - 例: =IF(OR(A1="完了", A1="済"), "OK", "未完了")

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **AND関数**: =AND(logical1, [logical2], ...)
  - すべての引数がTRUEの場合にTRUEを返す
  - 1つでもFALSEがあればFALSEを返す
- **OR関数**: =OR(logical1, [logical2], ...)
  - 1つでもTRUEがあればTRUEを返す
  - すべてFALSEの場合にFALSEを返す
- IFと組み合わせて複数条件の判定に使用するのは標準的な使用方法

**【参考文献】**:
- Microsoft Support: "AND 関数" (https://support.microsoft.com/ja-jp/office/5f19b2e8-e1df-4408-897a-ce285a19e9d9)

### 検証項目7: VLOOKUP関数

**【記載内容】**
- 書式: =VLOOKUP(検索値, 範囲, 列番号, 検索方法)
- 引数:
  - 検索値: 探したい値
  - 範囲: データが入っている表全体（1列目に検索値がある必要あり）
  - 列番号: 取得したい列が範囲の何列目か（1から始まる）
  - 検索方法: FALSE（完全一致）またはTRUE（近似値）
- 注意点: 検索値は範囲の1列目になければならない

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **VLOOKUP関数**: =VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
  - lookup_value: 検索する値
  - table_array: 検索範囲（1列目に検索値が必要）
  - col_index_num: 返す値がある列番号（1から始まる）
  - range_lookup: 
    - FALSE（0）: 完全一致
    - TRUE（1）: 近似一致（昇順ソート必須）
- **重要な制約**: VLOOKUPは検索範囲の**1列目（左端）**からのみ検索可能
- 検索値が見つからない場合は#N/Aエラーを返す（記載通り）

**【参考文献】**:
- Microsoft Support: "VLOOKUP 関数" (https://support.microsoft.com/ja-jp/office/0bbc8083-26fe-4963-8ab8-93a18ad188a1)

### 検証項目8: XLOOKUP関数

**【記載内容】**
- 用途: VLOOKUPの進化版（Microsoft 365、Excel 2021以降）
- 書式: =XLOOKUP(検索値, 検索範囲, 戻り範囲)
- VLOOKUPとの違い:
  - 列番号の指定が不要
  - 検索値が左端になくてもOK
  - 右から左への検索も可能
  - 検索値が見つからない場合の代替値を指定可能

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **XLOOKUP関数**: Excel 2021およびMicrosoft 365で導入された新しい検索関数
- **構文**: =XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])
- **VLOOKUPに対する改善点**:
  - 列番号不要（戻り範囲を直接指定）
  - 左方向検索可能（VLOOKUPの制約を解消）
  - デフォルトで完全一致
  - [if_not_found]引数で未検出時の代替値を指定可能
  - 水平・垂直検索の両方に対応
- **利用可能バージョン**: Microsoft 365、Excel 2021以降（記載通り）

**【参考文献】**:
- Microsoft Support: "XLOOKUP 関数" (https://support.microsoft.com/ja-jp/office/b7fd680e-6d10-43e6-84f9-88eae8bf5929)

### 検証項目9: INDEX + MATCH関数

**【記載内容】**
- 用途: VLOOKUPより柔軟な検索（左方向検索、複数条件検索が可能）
- 書式: =INDEX(取得したい列, MATCH(検索値, 検索列, 0))
- 例: =INDEX(B2:B5, MATCH(1002, A2:A5, 0))

**【検証結果】**: ✅ **正確**

**【根拠】**:
- **INDEX関数**: =INDEX(array, row_num, [column_num])
  - 配列内の指定された行・列の値を返す
- **MATCH関数**: =MATCH(lookup_value, lookup_array, [match_type])
  - 検索値が配列内の何番目にあるかを返す
  - match_type = 0: 完全一致
- **INDEX + MATCH の組み合わせ**:
  - MATCHで位置を特定し、INDEXでその位置の値を取得
  - VLOOKUPの制約（左端からの検索のみ）を回避可能
  - 列の追加・削除に強い（列番号ではなく範囲を指定するため）
  - 大量データでVLOOKUPより高速

**【参考文献】**:
- Microsoft Support: "INDEX 関数" (https://support.microsoft.com/ja-jp/office/a5dcf0dd-996d-40a4-a822-b56b061328bd)
- Microsoft Support: "MATCH 関数" (https://support.microsoft.com/ja-jp/office/e8dffd45-c762-47d6-bf89-533f4a37673a)

## 3. グラフ・ピボットテーブル

### 検証項目12: グラフの種類と用途

**【記載内容】**
- 縦棒グラフ: 項目間の比較
- 折れ線グラフ: 時系列の推移
- 円グラフ: 構成比率の表示
- 面グラフ: 累積推移の可視化
- 散布図: 2変数の相関関係
- ヒートマップ: 数値の大小を色で表現

**【検証結果】**: ✅ **正確**

**【根拠】**:
- Excelには以下の主要グラフタイプが存在:
  - **縦棒グラフ（Column Chart）**: カテゴリ間の比較に最適
  - **折れ線グラフ（Line Chart）**: 時間経過に伴う変化を表示
  - **円グラフ（Pie Chart）**: 全体に対する各要素の割合を表示
  - **面グラフ（Area Chart）**: 時間経過に伴う変化の量を強調
  - **散布図（Scatter Chart）**: 2つの変数間の関係性を表示
  - **ヒートマップ**: 条件付き書式を使用して数値の大小を色で視覚化
- 各グラフの用途は標準的なデータ可視化のベストプラクティスに準拠

  1. **近接（Proximity）**
  2. **整列（Alignment）**
  3. **反復（Repetition）**
  4. **コントラスト（Contrast）**
- **余白（White Space）**: デザインの重要な要素として広く認知
  - 30～40%の余白: デザイナーが推奨する実践的な目安
